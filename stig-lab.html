STIG Workflow

DoD STIGs (Security Technical Implementation Guides) are configuration standards published by DISA. They define specific security controls, hardening requirements, and verification methods for DoD and defense-adjacent systems. Each finding is classified CAT I (critical), CAT II (medium), or CAT III (low).

01
STIG Download & Scope
Download the Windows 11 STIG from DISA. Map STIG ID WN11-00-000105 to the specific control: SMBv1 client driver enablement. Define scope and target endpoint.
02
Finding Assessment
Assess the current state against each STIG requirement. Document the finding: open or not a finding. Classify by CAT severity. Record verification method used.
03
PowerShell Remediation
For each open finding, write and execute a PowerShell remediation script. Apply registry changes, service configurations, or Group Policy settings. Verify the fix.
04
Evidence Package
Capture before/after screenshots, command output, and registry state. Assemble the audit-ready evidence package. Documents the full chain from finding to remediation to verification.

STIG Findings — WN11-00-000105

Each finding below follows the DISA STIG documentation format: STIG ID, severity classification, check procedure, fix text, and PowerShell remediation.

CAT I
WN11-00-000105
SMBv1 Client Driver Must Be Disabled
The SMBv1 protocol is an outdated and insecure file-sharing protocol associated with critical vulnerabilities including EternalBlue (MS17-010) — exploited by WannaCry and NotPetya ransomware. The SMBv1 client driver (mrxsmb10) must be disabled to prevent exploitation of these known attack vectors.
Fix: Disable mrxsmb10 service and set start type to disabled via registry
✓ Remediated
Verification Method
Registry key: HKLM:\SYSTEM\CurrentControlSet\Services\mrxsmb10 — Start value must be 4 (Disabled). Verified via Get-ItemProperty and service state query.
CAT II
WN11-00-000110
SMBv1 Server Must Be Disabled
The SMBv1 server component must also be explicitly disabled. Even with the client driver disabled, the server-side SMBv1 listener can still accept inbound SMBv1 connections. Both client and server components must be independently verified and disabled.
Fix: Set-SmbServerConfiguration -EnableSMB1Protocol $false
✓ Remediated
Verification Method
PowerShell: Get-SmbServerConfiguration | Select EnableSMB1Protocol — must return False. Confirmed no SMBv1 listening state post-remediation.
CAT II
WN11-CC-000038
Autoplay Must Be Disabled for All Drives
Autoplay on removable media and non-system drives must be disabled. Autoplay executes code automatically when media is inserted — a common vector for physical access attacks and malware delivery via USB or optical media.
Fix: Group Policy — Turn off Autoplay for all drives
✓ Remediated
Verification Method
Registry: HKLM:\SOFTWARE\Policies\Microsoft\Windows\Explorer — NoDriveTypeAutoRun value must be 255 (0xff).
CAT III
WN11-00-000015
Windows 11 Version Must Be Current
The system must be running a supported version of Windows 11 with current security updates applied. End-of-support OS versions lack patches for disclosed vulnerabilities. Version verification is required as part of every STIG baseline assessment.
Fix: Windows Update — ensure current feature release and patch cycle
✓ Verified Compliant
Verification Method
PowerShell: (Get-WmiObject Win32_OperatingSystem).Version — Verified against DISA STIG supported version list.

PowerShell Remediation Scripts

Primary remediation for WN11-00-000105 — SMBv1 client driver disable and verification.

# STIG WN11-00-000105 — Disable SMBv1 Client Driver # Severity: CAT I | Date: 2024 | Verified: Pass # Step 1: Disable mrxsmb10 service (SMBv1 client driver) Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\mrxsmb10" ` -Name "Start" -Value 4 -Type DWord # Step 2: Disable SMBv1 server-side protocol Set-SmbServerConfiguration -EnableSMB1Protocol $false -Force # Step 3: Verify — SMBv1 client driver Get-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\mrxsmb10" | ` Select-Object Start # Expected output: Start : 4 # Step 4: Verify — SMBv1 server state Get-SmbServerConfiguration | Select-Object EnableSMB1Protocol # Expected output: EnableSMB1Protocol : False
