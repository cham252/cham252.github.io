Key Takeaways
01
Source-compiled malware bypasses package managers
apt-get said vsftpd was not installed — because it wasn't installed through apt. The backdoored binary lived in /usr/sbin and was managed by xinetd. File integrity monitoring (AIDE, Tripwire) would catch this regardless of installation method. Package manager evidence is a starting point, not a conclusion.
02
IOCs sat in auth.log for months before detection
Failed logins and SSH daemon bind errors from December 2025 through May 2026 were present in the logs the entire time. A SIEM with real-time log ingestion and alerting on authentication anomalies would have fired on day one. The NOC caught this via traffic spikes — the last line of defense, not the first.
03
Old systems use old service managers
Linux 2.6.X does not have systemctl or the service command. Containment and eradication commands that work on modern Ubuntu break on this system. Always check /etc/xinetd.d/ and /etc/inetd.conf on legacy targets when standard service management fails. Know the environment before you act.
04
Four Critical CVEs, one decade-old server
CVE-2011-2523 (2011), CVE-2007-2447 (2007), CVE-2010-2075 (2010), CVE-2020-1938. All publicly known. All still live. Weekly vulnerability scanning with CVSS 7.0+ alerting catches every one of these automatically. The cost of not scanning is this engagement.
05
AI is a speed multiplier — not a replacement for tools
Parsing a 170-second Nmap output for CVEs took seconds with AI. But the Metasploit session, the Wireshark decode, the auth.log review, and the netstat post-removal check are what made those findings real and submittable. AI generates the lead. The tool closes it. That order matters.
