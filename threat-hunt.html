Hunt Methodology

Hypothesis: An employee may be attempting to bypass corporate network controls using the Tor Browser, enabling exfiltration or anonymous communication outside of monitored channels. Hunt across DeviceFileEvents, DeviceProcessEvents, and DeviceNetworkEvents to confirm or deny.

01
DeviceFileEvents
Installation Detection
Queried file creation events for Tor-related filenames and installer patterns. KQL search for tor-browser-windows confirmed the silent installer was written to disk at 07:59 AM.
02
DeviceProcessEvents
Execution Tracing
Hunted for tor.exe process creation and parent process chains. Confirmed tor.exe launched from the extracted Tor Browser directory. Captured full process command line including /S silent install flag.
03
DeviceNetworkEvents
C2 Connection Confirmed
Queried outbound connections from tor.exe process. Confirmed live connection to 140.235.237.13:9001 — a known Tor relay node. Port 9001 is the standard Tor relay port.
// 02

Attack Timeline

07:59 AM — DeviceFileEvents
Silent Tor Browser Installer Written to Disk
File tor-browser-windows-x86_64-portable-14.5.8.exe created on target endpoint. The /S flag confirms silent (no-prompt) execution — a deliberate attempt to avoid user-visible installation dialogs and reduce detection probability.
08:00–08:09 AM — DeviceFileEvents
Tor Browser Files Extracted to Disk
Multiple Tor Browser component files extracted from the portable installer: firefox.exe, nssckbi.dll, tor.exe, and supporting libraries. File write events span nine minutes — consistent with a full portable browser extraction. All files written to a local user directory.
08:17–08:24 AM — DeviceProcessEvents
tor.exe Process Launched
tor.exe process created at 08:17 AM. Parent process chain: installer → firefox.exe (Tor Browser shell) → tor.exe. Process ran continuously for 7 minutes before network connections were established, consistent with Tor circuit initialization time.
08:24 AM — DeviceNetworkEvents
Live Outbound Connection to Tor Relay Confirmed
Outbound TCP connection from tor.exe to 140.235.237.13 on port 9001 — a publicly listed Tor relay node. Port 9001 is the standard Tor OR (onion routing) port used for relay-to-relay encrypted traffic. Connection confirmed active. Corporate proxy was bypassed — traffic did not pass through monitored channels.

KQL Queries Used

Three tables. One confirmed compromise.

// Stage 1 — Detect installer write (DeviceFileEvents) DeviceFileEvents | where FileName contains "tor" or FileName contains "firefox" | where ActionType == "FileCreated" | project Timestamp, DeviceName, FileName, FolderPath, InitiatingProcessFileName
// Stage 2 — Trace process execution (DeviceProcessEvents) DeviceProcessEvents | where FileName == "tor.exe" | project Timestamp, DeviceName, FileName, ProcessCommandLine, InitiatingProcessFileName, InitiatingProcessCommandLine
// Stage 3 — Confirm C2 connection (DeviceNetworkEvents) DeviceNetworkEvents | where InitiatingProcessFileName == "tor.exe" | where RemotePort == 9001 | project Timestamp, DeviceName, RemoteIP, RemotePort, RemoteUrl, InitiatingProcessFileName, ActionType

Remediation Steps

  • 1
    Isolate the endpoint via MDE network isolation — contain the device while investigation is active to prevent further exfiltration or lateral movement.
  • 2
    Terminate tor.exe and all associated processes (firefox.exe Tor shell). Kill process tree — not just the child process.
  • 3
    Delete Tor Browser files from all extraction directories. Search for tor.exe, tor-browser-windows-*.exe, and the full browser folder recursively.
  • 4
    Block Tor relay IPs and port 9001 at the firewall and proxy. Add 140.235.237.13 and all known public Tor relay exit nodes to the block list.
  • 5
    Review user activity for 30 days preceding the incident — email, USB, file transfers, and any outbound connections to Tor-adjacent infrastructure.
  • 6
    Escalate to HR and Legal if insider threat is confirmed. Preserve evidence chain-of-custody for potential disciplinary or legal action.
  • 7
    Update detection rules — add MDE custom detection for tor.exe, tor-browser-*.exe file creation events, and port 9001 outbound connections.
