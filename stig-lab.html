STIG Workflow

DoD STIGs (Security Technical Implementation Guides) are published by DISA. They define hardening requirements and verification methods for DoD and defense-adjacent systems. Each finding is classified CAT I (critical), CAT II (medium), or CAT III (low).

01
Download & Scope
Download STIG from DISA. Map STIG ID to control. Define scope and target endpoint.
02
Finding Assessment
Assess current state. Document open findings. Classify by CAT severity. Record verification method.
03
PowerShell Remediation
Write and execute remediation scripts. Apply registry changes, service configs, or GPO settings. Verify the fix.
04
Evidence Package
Capture before/after screenshots and command output. Assemble audit-ready evidence package.

STIG Findings — WN11-00-000105

Each finding follows the DISA STIG format: STIG ID, severity classification, check procedure, fix text, and PowerShell remediation.

CAT I
WN11-00-000105
SMBv1 Client Driver Must Be Disabled
SMBv1 is associated with EternalBlue (MS17-010) — exploited by WannaCry and NotPetya. The mrxsmb10 driver must be disabled to close this attack vector.
✓ Remediated
Verification
Registry key: HKLM:\SYSTEM\...\Services\mrxsmb10 — Start value must be 4 (Disabled).
CAT II
WN11-00-000110
SMBv1 Server Must Be Disabled
SMBv1 server component must be explicitly disabled. Even with client driver off, the server-side listener can still accept inbound SMBv1 connections. Both components require independent verification.
✓ Remediated
Verification
PowerShell: Get-SmbServerConfiguration | Select EnableSMB1Protocol — must return False.
CAT II
WN11-CC-000038
Autoplay Must Be Disabled
Autoplay on removable media and non-system drives must be disabled. Autoplay executes code on media insertion — common vector for physical access attacks and USB malware delivery.
✓ Remediated
Verification
Registry: NoDriveTypeAutoRun value must be 255 (0xff).
CAT III
WN11-00-000015
Windows 11 Version Must Be Current
System must run a supported Windows 11 version with current security updates. End-of-support OS versions lack patches for disclosed vulnerabilities.
✓ Verified Compliant
Verification
PowerShell: (Get-WmiObject Win32_OperatingSystem).Version — verified against DISA STIG supported version list.

PowerShell Remediation Scripts

Primary remediation for WN11-00-000105 — SMBv1 client driver disable and verification.

# STIG WN11-00-000105 — Disable SMBv1 Client Driver # Severity: CAT I | Verified: Pass # Step 1: Disable mrxsmb10 service (SMBv1 client driver) Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\mrxsmb10" ` -Name "Start" -Value 4 -Type DWord # Step 2: Disable SMBv1 server-side protocol Set-SmbServerConfiguration -EnableSMB1Protocol $false -Force # Step 3: Verify — SMBv1 client driver state Get-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\mrxsmb10" | ` Select-Object Start # Expected: Start : 4 # Step 4: Verify — SMBv1 server state Get-SmbServerConfiguration | Select-Object EnableSMB1Protocol # Expected: EnableSMB1Protocol : False
