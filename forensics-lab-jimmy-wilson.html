Jimmy Wilson and Billy Bob are lower-level realtors at SQue Worldwide Realty who share a corporate workstation. During a routine IT maintenance cycle, personnel discovered potentially disturbing items and files on the device. The machine was turned over to the digital forensics team for examination. This lab documents the examination methodology, evidence recovered, and findings presented to HR.

02 Examination Methodology

The examination followed a structured digital forensics workflow consistent with industry best practices for corporate endpoint investigations. All steps were performed on a forensic copy of the original evidence to preserve integrity.

  • 01

    Environment Setup

    Logged into ProxMox virtual infrastructure. Booted the Windows 11 VM containing the Autopsy forensic toolkit. Confirmed VM integrity and access credentials prior to starting examination.

  • 02

    Evidence Intake and Hashing

    Located 2020JimmyWilson.E01 in the Documents → Forensics folder. Documented file name, path, file size, and creation timestamp prior to ingestion. Verified MD5 and SHA-1 hash values upon Autopsy ingest to confirm evidence integrity.

  • 03

    Case Creation and Image Ingest

    Created a new case in Autopsy with examiner identification metadata. Added the E01 as the primary data source. Allowed full ingest to complete with all available ingest modules active. Ingest duration: approximately 1 hour.

  • 04

    Systematic Evidence Examination

    Examined six key artifact categories in the Autopsy tree panel: images, documents (HTML and Office), email messages, web history, web search, and OS accounts. Each category was reviewed for evidence of wrongdoing by either subject.

  • 05

    Bookmarking and Evidence Tagging

    All items of forensic interest were bookmarked in Autopsy using appropriate classification tags. Metadata captured for each item included: file name, full path, created time, file size, and hash value.

  • 06

    Report Generation

    Generated the Autopsy HTML report from all bookmarked items. Supplemented with the formal examiner report using the provided organizational template for HR department presentation.

03 Evidence Examination Areas

Autopsy's automated ingest modules and manual review covered six artifact categories. Each area was examined for evidence of policy violations, inappropriate content, or other items of interest for the HR investigation.

// 01
Images
Reviewed all image files recovered from the filesystem including deleted and carved items. Flagged any inappropriate, illegal, or policy-violating visual content.
// 02
Documents
Examined HTML files and Microsoft Office documents (.docx, .xlsx, .pptx, .pdf) for sensitive or incriminating content, proprietary data, or external communications.
// 03
Email Messages
Analyzed email artifacts including locally stored messages, sent items, and drafts. Identified communications between subjects and any external parties relevant to the investigation.
// 04
Web History
Reconstructed browser history from Chrome, Firefox, and Edge artifacts. Documented visited URLs, timestamps, and frequency of access to flagged domains.
// 05
Web Search
Extracted search queries from browser artifacts and keyword history. Identified search terms used by subjects that are relevant to the investigation.
// 06
OS Accounts
Examined Windows user account data including SAM registry, login history, account creation dates, and privilege levels for all accounts on the shared workstation.
04 Forensic Workflow
A

Acquire — Evidence Intake

E01 image received from IT department. File size, location, and pre-ingest hash values documented. Chain of custody established.

B

Preserve — Hash Verification

MD5 and SHA-1 hashes generated at ingest and compared to confirm image integrity. Forensic examination performed on Autopsy working copy only.

C

Analyze — Artifact Examination

Six artifact categories systematically examined. Suspicious and relevant items bookmarked with category tags. Screenshots taken with metadata visible.

D

Document — Report Generation

Autopsy HTML report generated from all tagged items. Examiner report written using organizational template. Findings structured for HR department audience.
